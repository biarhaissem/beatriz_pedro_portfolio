CupCakePop

Full Stack Project

Site para venda de cupcakes construído em várias etapas.

Desafio & Objetivo

Este projeto foi muito desafiador e complexo pois eu não tinha nenhuma experiência prévia: conforme eu ia aprendendo os conceitos, já desenvolvia o trabalho na prática.

Além das aulas da faculdade, estudei, pesquisei e consultei diversas fontes para obter o máximo possível de informação para organizar o cronograma de trabalho.

O principal objetivo deste projeto foi aprender como são as etapas de desenvolvimento de uma aplicação web.

01. Benchmarking

Pesquisa prévia para imersão no universo dos sites de venda de cupcakes.

Essa etapa é imprescindível para o desenvolvimento de qualquer projeto pois ajuda na elaboração das funcionalidades, layout e ideias de modo geral.

02. Wireframes

Primeiros esboços para possibilidades do layout do site.

Sempre gostei de fazer um rascunho antes de realizar qualquer atividade, mas nessa época, eu nem sabia o que era um "wireframe", rsrsrs!

03. Wix Website

Website desenvolvido através da plataforma Wix, com customização de template.

Eu já havia utilizado a plataforma para construção de sites no passado, muito antes de cursar a faculdade.

A plataforma Wix é muito interessante pois podemos escolher templates prontos e gratuitos para customizarmos de acordo com o nosso objetivo.

04. New Website

Website criado com HTML, CSS, JavaScript e Bootstrap.

Nesta fase, tive mais dificuldades pois foi a minha primeira experiência de desenvolvimento web.

Não foi fácil, mas com muito esforço, dedicação, estudo e mil perguntas para meus colegas mais experientes, o esboço foi tomando forma.

{{ item.title }}

05. Angular Website

Criação do Website utilizando o framework Angular e a linguagem TypeScript.

Considero esta etapa a mais difícil de todo o curso de Análise e Desenvolvimento de Sistemas.

Sofri muito, pois eu não sabia por onde começar. A primeira experiência com o framework Angular ficará guardada para sempre em minha memória.

Eu queria realmente entender como a arquitetura poderia ser organizada, mas como eu era inexperiente, tive que ter muita paciência e resiliência para encontrar a luz no final do túnel...

{{ item.title }}

06. Front-end & Back-end

Etapa final de desenvolvimento, utilizando o framework Angular e a linguagem TypeScript no Front-end e, o framework Laravel e a linguagem PHP no Back-end.

Nessa etapa final, eu já estava mais tranquila e achei muito interessante essa organização onde o front-end e o back-end ficam separados.

Estudando ambas as estruturas, front e back, percebi o quanto facilita o desenvolvimento e aumenta a segurança do projeto trabalhar com essa separação.

Construí duas versões finais: uma com Angular e Laravel, bem separadinha e, outra, apenas com o Laravel para visualização do "CRUD" (Create, Read, Update, Delete), utilizando o Blade.

Depois de todo o esforço, dedicação, lágrimas e insistência, vem a melhor parte: funcionou!

Pra mim, a sensação de ver que os dados registrados no cadastro do front-end "apareciam" nos registros do back-end, simultaneamente ao preenchimento do Banco de Dados e da API com Json, foi MÁGICO!
